# قرارات مشروع ConsuTrain

آخر مراجعة: 2026-07-12

## حالات القرار

- **معتمد:** يوجّه التنفيذ الحالي.
- **مستبدل:** قرار تاريخي لم يعد يوجّه التنفيذ، مع حفظ سبب الاستبدال.
- **قيد المراجعة:** لم يُحسم أو يحتاج تحققًا إضافيًا.

## القرارات المعتمدة

### الاستضافة عبر GitHub Pages والنطاق عبر Cloudflare

- التاريخ: موثق في السجل السابق خلال مايو 2026، دون يوم اعتماد دقيق.
- الحالة: معتمد.
- السبب: يناسب بنية الموقع الثابتة، ويتيح نشرًا مباشرًا من المستودع مع إدارة DNS وطبقة حماية طرفية.
- الأثر: يبقى `CNAME` مرجع النطاق، وتُراجع إعدادات GitHub Pages وCloudflare عند أي تغيير في النشر أو الحماية.

### اعتماد العربية والفرنسية في بنية الموقع

- التاريخ: التوسع الفرنسي موثق بين 2026-06-14 و2026-06-19.
- الحالة: معتمد.
- السبب: خدمة الجمهور المستهدف باللغتين مع الحفاظ على تجربة متسقة.
- الأثر: العربية في البنية الأساسية، والفرنسية تحت `fr/` مع أجزاء وبيانات وصفحات مقابلة؛ يجب الحفاظ على التكافؤ عندما يتوفر المحتوى باللغتين.

### اعتماد النماذج الداخلية بدل الأدوات الخارجية المضمنة

- التاريخ: 2026-06-14، وتثبيت مسار الاستقبال موثق في 2026-06-15 و2026-06-19.
- الحالة: معتمد.
- السبب: توحيد تجربة الزائر وتقليل الاعتماد على نموذج طرف ثالث وإتاحة تحكم أفضل في البيانات والتوسع.
- الأثر: تستخدم الاستشارات وطلبات الأسعار والملاحظات صفحات داخلية، ولا يعاد إدخال Tally.

### اعتماد واتساب سريع ونموذج منظم داخل الموقع

- التاريخ: غير موثق بدقة.
- الحالة: معتمد.
- السبب: واتساب مناسب للاتصال السريع، بينما يحفظ النموذج المنظم معلومات الطلب اللازمة للمتابعة.
- الأثر: يُحافظ على الخيارين دون استبدال النموذج برابط مراسلة فقط، ودون وضع رابط غير مؤكد.

### اعتماد دليل حي للمنصة بالعربية والفرنسية

- التاريخ: الدليل العربي موثق في 2026-07-11، والفرنسي في 2026-07-12 وفق سجل Git.
- الحالة: معتمد.
- السبب: توفير عرض محدث للخدمات والدورات والأدوات والموارد بدل الاعتماد على ملف ثابت فقط.
- الأثر: تبقى صفحتا `consutrain-guide.html` و`fr/consutrain-guide.html` واجهتي الدليل الحي.

### اعتماد JSON مصدرًا لمحتوى دليل المنصة

- التاريخ: 2026-07-11 و2026-07-12 وفق تنفيذ الدليل والمولد.
- الحالة: معتمد.
- السبب: منع تكرار المحتوى بين صفحة الويب وملفات PDF وتسهيل التحديث الثنائي.
- الأثر: يُحدّث المحتوى في `assets/data/platform-guide.json` و`assets/data/fr-platform-guide.json`، وتبقى بيانات الاتصال في `assets/data/platform-contact.json`.

### اعتماد نسختي PDF عربية وفرنسية مولدتين آليًا

- التاريخ: 2026-07-12.
- الحالة: معتمد.
- السبب: إتاحة دليل قابل للتنزيل ومتسق مع البيانات الحية في اللغتين.
- الأثر: يشغّل المولد نفسه النسختين، وتُعامل ملفات PDF والـmanifest كمخرجات لا كمصادر تحرير يدوي.

### اعتماد GitHub Actions لتحديث أحدث ملفات PDF

- التاريخ: 2026-07-12.
- الحالة: معتمد.
- السبب: ضمان إعادة التوليد عند تغير بيانات الدليل أو المولد أو الأصول الداخلة في العملية.
- الأثر: ينفذ `.github/workflows/update-platform-guide-pdfs.yml` التوليد والتحقق، وينشئ commit آليًا للمخرجات عند وجود فرق؛ يجب مراجعة النتيجة وسحبها قبل تعديل متوازٍ.

### إبقاء روابط PDF ثابتة مع تحديث المحتوى

- التاريخ: 2026-07-12.
- الحالة: معتمد.
- السبب: منع كسر الروابط المنشورة أو المحفوظة عند كل إصدار.
- الأثر: تبقى أسماء الملفين داخل `resources/downloads/profile/` ثابتة، بينما تتغير البصمات والإصدار في manifest.

### اعتماد الموارد المجانية مدخلًا قبل الخدمات المدفوعة

- التاريخ: موثق منذ 2026-05-06.
- الحالة: معتمد.
- السبب: تمكين الزائر من تجربة قيمة عملية وفهم المنهج قبل طلب خدمة مدفوعة.
- الأثر: تُربط الموارد بصفحات شرح ومسارات خدمات ذات صلة، ويظل التنزيل المجاني جزءًا أساسيًا من رحلة الزائر.

### اعتماد تدريبات مجانية قصيرة مع تقييم وشهادة رقمية

- التاريخ: مسار الشهادات موثق منذ مايو 2026، وتوسعت التدريبات حتى يونيو 2026.

- الأثر: يتضمن كل تدريب مكتمل دروسًا وتقييمًا نهائيًا وطلب شهادة إتمام رقمية، مع منع التكرار وعدم الادعاء بالاعتماد الرسمي.

### عدم تضمين قسم المصادر في مقالات الموقع إلا بطلب صريح

- التاريخ: غير موثق بتاريخ دقيق.

- الأثر: لا يضاف قسم مصادر تلقائيًا إلى المقالات العربية أو الفرنسية.

### اعتماد بنية قابلة للتوسع للخدمات والدورات والموارد

- التاريخ: مبدأ متكرر في قرارات الموارد منذ مايو 2026 والتوسع الفرنسي في يونيو 2026.
- الحالة: معتمد.
- السبب: تجنب الصفحات والمسارات الخاصة التي يصعب إدارتها عند نمو المحتوى.
- الأثر: تستخدم أقسامًا وفهارس ومجلدات واضحة وملفات بيانات عند ملاءمتها، ويُراعى التوسع الثنائي من البداية.

### عدم تنفيذ Codex لعمليات commit أو push

- التاريخ: غير موثق بتاريخ اعتماد سابق.
- الحالة: معتمد.
- السبب: إبقاء قرار تسجيل التغييرات ونشرها بيد المستخدم، وتمكينه من مراجعة diff أولًا.
- الأثر: يقتصر عمل Codex على التعديل والفحص والتقرير، ولا ينفذ `commit` أو `push`.

## أعمال وقرارات سابقة محفوظة

شملت المراحل السابقة قرارات تفصيلية حول مكتبة المعرفة، مسارات المهارات والإدارة بالأهداف، الموارد المهنية وقوالبها، تبسيط الهيدر، أزرار المشاركة والعودة، المصطلحات الفرنسية، توطين صفحات الخدمات والدورات والأدوات، حماية الإدارة، وتحسين PWA. ما زال أثر هذه القرارات ظاهرًا في الملفات الفعلية، لكن تفاصيل التنفيذ اليومية والقرارات المكررة لم تعد جزءًا من سجل القرار الحالي. عند الحاجة إلى تاريخ دقيق لتغيير قديم، يُرجع إلى سجل Git والملف المتأثر بدل اعتبار هذه الخلاصة سجلًا زمنيًا كاملًا.

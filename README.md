# ConsuTrain MVP

ConsuTrain منصة ويب للتعلم الإداري والتدريب والخدمات الاستشارية والأدوات والموارد العملية. يعمل المشروع كموقع ثابت ثنائي اللغة، بالعربية والفرنسية، ويشمل PWA ونماذج داخلية وتدريبات مجانية بشهادات إتمام رقمية ودليلًا حيًا للمنصة.

## التقنية

- HTML وCSS وJavaScript دون إطار تطبيق رئيسي.
- ملفات JSON لفهارس المحتوى وبيانات الدليل.
- Python لتوليد دليلي PDF.
- GitHub Pages للنشر وCloudflare للنطاق والطبقة الطرفية.
- GitHub Actions لتحديث ملفات PDF المولدة.
- n8n لمسارات الأتمتة المتقدمة، ومنها الشهادات الرقمية.

## التشغيل محليًا

من جذر المستودع شغّل خادمًا محليًا بدل فتح الصفحات مباشرة، حتى تعمل المسارات والأجزاء المشتركة وطلبات الملفات بصورة صحيحة:

```powershell
py -m http.server 8000 --bind 127.0.0.1
```

ثم افتح `http://127.0.0.1:8000/`. يمكن استخدام `python` بدل `py` إذا كان هو الأمر المتاح في البيئة.

## البنية الأساسية

- `fr/`: الصفحات الفرنسية ونظائر الأقسام العامة.
- `partials/`: الهيدر والفوتر المشتركان بالعربية والفرنسية.
- `assets/css/` و`assets/js/`: الأنماط والسلوك المشترك.
- `assets/data/`: فهارس المقالات والمهارات وبيانات دليل المنصة.
- `learn/`: المقالات ومسارات التعلم والتدريبات المجانية.
- `courses/`: الدورات والورش.
- `services/`: صفحات الخدمات ونموذج الاستشارة.
- `resources/` و`templates/`: الموارد المجانية وصفحات شرح القوالب والتنزيلات.
- `tools/`: الأدوات الإدارية والتفاعلية.
- `admin/`: واجهات الإدارة الداخلية.
- `scripts/`: أدوات التوليد، ومنها مولد PDF.
- `.github/workflows/`: أعمال GitHub Actions.

## الهيدر والفوتر المشتركان

تستخدم الصفحات العربية `partials/header.html` و`partials/footer.html`، وتستخدم الفرنسية `partials/fr-header.html` و`partials/fr-footer.html`. يتولى `assets/js/includes.js` تحميل الأجزاء.

عند إنشاء صفحة متداخلة، اضبط `data-root` حسب عمقها من الجذر، وراجع روابط CSS وJavaScript والصور وسمات breadcrumb. لا تنسخ الهيدر أو الفوتر كاملًا داخل صفحة جديدة.

## ملفات البيانات المهمة

- `assets/data/articles.json`: فهرس المقالات العربية.
- `assets/data/fr-articles.json`: فهرس المقالات الفرنسية.
- `assets/data/glossary.json`: بيانات المصطلحات.
- `assets/data/soft-skills.json`: محتوى المهارات الناعمة.
- `assets/data/platform-guide.json`: محتوى دليل المنصة العربي.
- `assets/data/fr-platform-guide.json`: محتوى دليل المنصة الفرنسي.
- `assets/data/platform-contact.json`: بيانات الاتصال والروابط المشتركة للدليل.
- `assets/data/platform-guide-pdf-manifest.json`: manifest مولد لإصدارات PDF، وليس مصدر تحرير يدوي.

## دليل المنصة وتوليد PDF

يعرض `consutrain-guide.html` الدليل العربي، ويعرض `fr/consutrain-guide.html` الدليل الفرنسي. لتحديث المحتوى، عدّل ملف JSON الخاص باللغة وبيانات الاتصال المشتركة عند الحاجة، ثم تحقق من الدليلين.

يولّد `scripts/generate_platform_guide_pdfs.py` الملفين التاليين بأسماء ثابتة:

- `resources/downloads/profile/ConsuTrain_Profile_and_Services_AR.pdf`
- `resources/downloads/profile/ConsuTrain_Profile_and_Services_FR.pdf`

التعليمات التفصيلية للتوليد اليدوي موجودة في [README-auto-guide-pdfs.md](README-auto-guide-pdfs.md). ويعمل `.github/workflows/update-platform-guide-pdfs.yml` تلقائيًا عند تغير مصادر الدليل المحددة. راجع نتيجة GitHub Actions واسحب commit الآلي قبل تعديل المصادر نفسها مرة أخرى؛ لا تعدّل PDF أو manifest يدويًا بالتوازي مع الأتمتة.

## Service Worker

الملف الرئيسي هو `sw.js`. عند تعديل ملف في precache أو إضافة ملف يجب تخزينه مسبقًا:

1. تحقق من صحة المسار ووجود الملف.
2. حدّث قائمة precache عند الحاجة.
3. غيّر اسم الكاش حتى يحصل الزائر على النسخة الجديدة.
4. اختبر التحميل والتحديث والعمل دون اتصال.

## إضافة محتوى

### مقالة

أنشئ الصفحة في مسار اللغة الصحيح، ثم أضف بياناتها إلى `assets/data/articles.json` أو `assets/data/fr-articles.json`. تحقق من الرابط، والصورة، والفهرسة، والظهور في صفحة المقالات. لا تضف قسم مصادر إلا إذا كان مطلوبًا صراحة.

### دورة أو تدريب

استخدم مجلدًا واضحًا وقابلًا للتوسع تحت `courses/` أو مسار التدريب المجاني المناسب، وحدّث الفهرس والروابط ذات الصلة. إذا كان المحتوى متاحًا باللغتين فأنشئ المقابل الفرنسي وتحقق من التنقل بينهما. لا تصف شهادة الإتمام بأنها اعتماد رسمي.

### خدمة

أنشئ صفحة الخدمة داخل `services/` ونظيرها تحت `fr/services/` عندما يتوفر المحتوى الفرنسي، ثم حدّث فهرس الخدمات وروابط الطلب. استخدم نموذج الموقع الداخلي ولا تضف نموذج طرف ثالث أو عنوان استقبال حساس داخل الوثائق.

## الأمان وGit

لا تضع مفاتيح أو رموز وصول أو بيانات دخول أو Webhooks أو بيانات شخصية حساسة في المستودع. استخدم إعدادات البيئة أو منصة الأتمتة لإدارة الأسرار.

Codex لا ينفذ `commit` أو `push`. يجب أن يراجع المستخدم `git diff` ونتائج الاختبارات ثم يقرر تسجيل التغييرات ونشرها.

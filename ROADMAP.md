# ROADMAP — ConsuTrain-MVP

## 1. Purpose of this file

This file defines the practical development roadmap for the ConsuTrain project.

It helps keep the project organized by clarifying:
- the current phase
- the next priorities
- what should be done now
- what should be postponed
- which project path each phase belongs to

This roadmap should be reviewed and updated after each major development milestone.

---

## 2. Project direction

ConsuTrain is being developed as a professional platform that combines:

- learning content
- practical tools
- consulting services
- AI-supported resources
- future courses and digital products

The current objective is to stabilize the MVP before expanding content, tools, courses, or monetization.

---

## 3. Current phase

Current phase:

**Phase 1 — MVP stabilization**

The website already has:
- main pages
- learning pages
- service pages
- tools pages
- shared header and footer
- PWA files
- project rules
- project status tracking
- decisions log

The focus now is to make sure the current version is stable, clear, and ready for gradual expansion.

---

# 6. Work that should not be started yet

The following work should be postponed until the MVP is stable:

- adding many new tools
- building paid products
- creating a member area
- adding backend technologies
- restructuring the whole design
- adding too many content pages at once
- launching monetization pages before improving service content

---

# 7. Standard testing rule before moving between phases

Before moving from one phase to another, the following must be checked:

- homepage works
- header loads correctly
- footer loads correctly
- navigation works
- internal links work
- mobile view works
- GitHub Pages version works
- PWA behavior is acceptable
- no important 404 errors appear
- no old cached header, CSS, or JavaScript files appear

# ROADMAP — ConsuTrain-MVP

## 1. Purpose of this file

This file defines the practical development roadmap for the ConsuTrain project.

It helps keep the project organized by clarifying:
- the current phase
- the next priorities
- what should be done now
- what should be postponed
- which project path each phase belongs to

This roadmap should be reviewed and updated after each major development milestone.

---

## 2. Project direction

ConsuTrain is being developed as a professional platform that combines:

- learning content
- practical tools
- consulting services
- AI-supported resources
- future courses and digital products

The current objective is to stabilize the MVP before expanding content, tools, courses, or monetization.

---

## 3. Current phase

Current phase:

**Phase 1 — MVP stabilization**

The website already has:
- main pages
- learning pages
- service pages
- tools pages
- shared header and footer
- PWA files
- project rules
- project status tracking
- decisions log

The focus now is to make sure the current version is stable, clear, and ready for gradual expansion.

# 6. Work that should not be started yet

The following work should be postponed until the MVP is stable:

- adding many new tools
- building paid products
- creating a member area
- adding backend technologies
- restructuring the whole design
- adding too many content pages at once
- launching monetization pages before improving service content

---

# 7. Standard testing rule before moving between phases

Before moving from one phase to another, the following must be checked:

- homepage works
- header loads correctly
- footer loads correctly
- navigation works
- internal links work
- mobile view works
- GitHub Pages version works
- PWA behavior is acceptable
- no important 404 errors appear
- no old cached header, CSS, or JavaScript files appear

# ROADMAP – ConsuTrain

## المرحلة الحالية

تم الانتهاء من تثبيت البنية الأساسية للموقع، وإعادة بناء الصفحات المحورية، وإضافة قسم الورش التطبيقية القصيرة.

## الأولويات القادمة

### الأولوية 1 – مراجعة صفحة الخدمات

الهدف:
تحسين صفحة الخدمات لتكون أكثر وضوحًا وتسويقًا، مع الحفاظ على الاختصار.

المهام:
- مراجعة ترتيب الخدمات.
- التأكد من وضوح الخدمات المدفوعة.
- تحسين نصوص البطاقات أو الأكورديون.
- التأكد من وجود دعوة واضحة للتواصل أو طلب عرض سعر.
- اختبار الصفحة على الهاتف.

الحالة:
قيد التخطيط.

---

### الأولوية 2 – ربط الخدمات بالدورات والورش

الهدف:
كل خدمة رئيسية يجب أن ترتبط بما يدعمها من دورات وورش وموارد.

أمثلة:
- خدمة التخطيط الاستراتيجي ترتبط بدورة التخطيط، دورة بطاقة الأداء، ورشة SMART، ورشة KPI.
- خدمة الجودة والآيزو ترتبط بدورة ISO 9001، IMS، ورشة عدم المطابقة.
- خدمة الحوكمة ترتبط بدورة الحوكمة، ورشة مصفوفة الصلاحيات.
- خدمة إدارة المخاطر ترتبط بورشة سجل المخاطر.

الحالة:
مخطط.

---

### الأولوية 3 – تحسين نظام التحويل Contact / Conversion

الهدف:
تسهيل انتقال الزائر من المحتوى إلى التواصل.

المهام:
- التأكد من ظهور زر واتساب سريع.
- التأكد من ظهور نموذج Tally أو رابط طلب استشارة.
- توحيد الدعوات في الصفحات:
  - سجل اهتمامك.
  - اطلب ورشة مخصصة.
  - اطلب عرض سعر.
  - تواصل معنا.
- مراجعة صفحة التواصل.

الحالة:
مخطط.

---

### الأولوية 4 – تنظيم المحتوى التعليمي والدورات

الهدف:
استكمال تنظيم الدورات والفيديوهات داخل قسم Courses.

المهام:
- مراجعة ترتيب الدورات.
- التأكد من ظهور الدورات الجديدة في الهيدر وصفحة الدورات.
- ربط الدورات بالمسارات العملية.
- استخدام مكتبة محتوى YouTube عند الحاجة.
- التفكير لاحقًا في صفحة تفصيلية لكل مسار.

الحالة:
مستمر.

---

### الأولوية 5 – تطوير صفحة الموارد المجانية

الهدف:
جعل الموارد المجانية أكثر تنظيمًا وقابلية للتصفح.

المهام:
- تصنيف الموارد حسب المجال.
- إبراز الموارد المرتبطة بالخدمات.
- إضافة موارد جديدة تدريجيًا.
- الحفاظ على تصميم البطاقات الحالي.
- تجنب فصل CSS خاص بالموارد ما لم تكن هناك ضرورة.

الحالة:
مخطط.

---

### الأولوية 6 – تطوير الأدوات التفاعلية

الهدف:
زيادة قيمة الموقع من خلال أدوات مجانية قابلة للاستخدام.

الأدوات الحالية أو المعتمدة:
- ومضة إدارية.
- MyTodo.
- أدوات أخرى ضمن قسم Tools.

المهام القادمة:
- تحسين ظهور ومضة إدارية في الصفحة الرئيسية.
- مراجعة زر المشاركة.
- تطوير أدوات مرتبطة بالخدمات مثل:
  - مولد هدف SMART.
  - نموذج بطاقة KPI.
  - مساعد سجل المخاطر.
  - مساعد مصفوفة الصلاحيات.

الحالة:
مخطط.

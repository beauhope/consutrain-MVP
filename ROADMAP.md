# ROADMAP — ConsuTrain-MVP

## 1. Purpose of this file

This file defines the practical development roadmap for the ConsuTrain project.

It helps keep the project organized by clarifying:
- the current phase
- the next priorities
- what should be done now
- what should be postponed
- which project path each phase belongs to

This roadmap should be reviewed and updated after each major development milestone.

---

## 2. Project direction

ConsuTrain is being developed as a professional platform that combines:

- learning content
- practical tools
- consulting services
- AI-supported resources
- future courses and digital products

The current objective is to stabilize the MVP before expanding content, tools, courses, or monetization.

## الأولويات القادمة

### الأولوية 1 – مراجعة صفحة الخدمات

الهدف:
تحسين صفحة الخدمات لتكون أكثر وضوحًا وتسويقًا، مع الحفاظ على الاختصار.

المهام:
- مراجعة ترتيب الخدمات.
- التأكد من وضوح الخدمات المدفوعة.
- تحسين نصوص البطاقات أو الأكورديون.
- التأكد من وجود دعوة واضحة للتواصل أو طلب عرض سعر.
- اختبار الصفحة على الهاتف.

الحالة:
قيد التخطيط.

---

### الأولوية 2 – ربط الخدمات بالدورات والورش

الهدف:
كل خدمة رئيسية يجب أن ترتبط بما يدعمها من دورات وورش وموارد.

أمثلة:
- خدمة التخطيط الاستراتيجي ترتبط بدورة التخطيط، دورة بطاقة الأداء، ورشة SMART، ورشة KPI.
- خدمة الجودة والآيزو ترتبط بدورة ISO 9001، IMS، ورشة عدم المطابقة.
- خدمة الحوكمة ترتبط بدورة الحوكمة، ورشة مصفوفة الصلاحيات.
- خدمة إدارة المخاطر ترتبط بورشة سجل المخاطر.

الحالة:
مخطط.

---

- التأكد من ظهور نموذج Tally أو رابط طلب استشارة.
- توحيد الدعوات في الصفحات:
  - سجل اهتمامك.
  - اطلب ورشة مخصصة.

## المسارات المستقبلية لمسار الشهادات الرقمية

بعد اكتمال دورة إصدار الشهادات الرقمية الأساسية، تشمل المسارات المستقبلية المقترحة ما يلي:

### 1. تحسين تجربة المستخدم عند التكرار

عند محاولة المتدرب إعادة الاختبار بنفس البريد ونفس التدريب، يجب عرض رسالة أوضح داخل الموقع، مثل:

"تم إصدار شهادة لهذا البريد في هذا التدريب سابقًا. يرجى مراجعة بريدك الإلكتروني."

مع إمكانية إضافة خيار لاحق لإعادة إرسال الشهادة بدل إصدار شهادة جديدة.

### 2. صفحة أو لوحة إدارة الشهادات

إنشاء صفحة أو لوحة داخلية لإدارة الشهادات، تشمل:

- البحث عن شهادة حسب البريد.
- البحث عن شهادة حسب اسم التدريب.
- البحث عن شهادة حسب رقم الشهادة.
- عرض حالة الشهادة: Pending / Processing / Sent / Failed.
- إعادة إرسال الشهادة يدويًا عند الحاجة.
- مراجعة الأخطاء إن حدثت.

### 3. صفحة تحقق من الشهادة

إنشاء صفحة تحقق عامة مستقبلية، مثل:

`verify.html`

أو:

`/verify`

تسمح بالتحقق من رقم الشهادة دون عرض ملفات الشهادات الخاصة مباشرة.

البيانات المعروضة يجب أن تكون محدودة، مثل:

- حالة الشهادة.
- اسم المتدرب.
- اسم التدريب.
- تاريخ الإصدار.
- رقم الشهادة.

### 4. دعم اختبارات وتدريبات متعددة

يمكن لأي اختبار جديد استخدام نفس Webhook ونفس مسار إصدار الشهادات، بشرط إرسال الحقول الأساسية، وأهمها:

- `trainingId`
- `trainingTitle`
- `fullName`
- `email`
- `score`
- `totalQuestions`
- `percentage`
- `passed`
- `submissionId`
- `certificateKey`

يجب أن يكون `trainingId` فريدًا لكل تدريب حتى يتم منع التكرار داخل نفس التدريب فقط، مع السماح للمتدرب بالحصول على شهادات في تدريبات مختلفة.

### 5. نقل n8n إلى بيئة مستضافة

في المرحلة الحالية يعمل n8n محليًا على جهاز Windows باستخدام Startup script.

لاحقًا، عند الإطلاق العام للمنصة، يمكن دراسة أحد الخيارين:

- n8n Cloud لسهولة التشغيل وتقليل الصيانة.
- VPS self-hosted لتقليل التكلفة والتحكم الكامل.

### 6. الانتقال التدريجي إلى بنية SaaS

عند تطوير حسابات العملاء ولوحاتهم الشخصية، يمكن الانتقال تدريجيًا إلى بنية أكثر توسعًا، مثل:

- Frontend عبر Vercel أو Render.
- قاعدة بيانات وحسابات عبر Supabase أو PostgreSQL.
- أتمتة عبر n8n Cloud أو VPS.
- الفيديوهات تبقى مستضافة على YouTube.
- الشهادات والملفات تحفظ لاحقًا في تخزين سحابي منظم.

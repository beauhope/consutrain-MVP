# خارطة طريق ConsuTrain

آخر مراجعة: 2026-07-12

لا تتضمن هذه الخارطة مواعيد غير معتمدة. الحالة الفعلية في `PROJECT-STATUS.md`، والقرارات الملزمة في `DECISIONS.md`.

## 1. مكتمل

### الأساس التقني والنشر

- موقع ثابت منشور عبر GitHub Pages ومربوط بالنطاق الرسمي عبر Cloudflare.
- بنية PWA تشمل manifest وService Worker وصفحة عدم اتصال.
- أجزاء مشتركة للهيدر والفوتر بالعربية والفرنسية مع تحميل مركزي.
- بنية عربية وفرنسية للصفحات العامة الأساسية.

### المحتوى والخدمات

- بوابات رئيسية للتعلم والدورات والخدمات والأدوات والموارد.
- مكتبة معرفة ومسارات تعلم إدارية متعددة.
- صفحات خدمات عربية وفرنسية وفهارس دورات باللغتين.
- مكتبة موارد مجانية وقوالب مشروحة مع تنزيلات منظمة حسب المجال.
- رادار أدوات الإدارة الرقمية وصفحات وتحليلات عربية وفرنسية أساسية.

### النماذج وتجربة التحويل

- نماذج داخلية بالعربية والفرنسية للاستشارة وطلب السعر والملاحظات.
- ربط النماذج العامة المفحوصة بمسار استقبال Google Apps Script.
- اعتماد رابط واتساب سريع إلى جانب النماذج المنظمة.
- إزالة الاعتماد الحالي على Tally من مسارات الزائر المعتمدة.

### التدريبات والشهادات

- أربعة تدريبات مجانية قصيرة بالعربية وأربعة مسارات مقابلة بالفرنسية.
- تقييم نهائي وطلب شهادة إتمام رقمية لكل تدريب مكتمل.
- مسار موحد لاستقبال طلبات الشهادات ومنع التكرار.
- لوحة داخلية للشهادات في وضع القراءة فقط وبنية إدارة قابلة للتوسع.

### دليل المنصة وPDF

- دليل ConsuTrain حي بالعربية والفرنسية.
- ملفات JSON منفصلة لمحتوى الدليل في اللغتين وملف موحد لبيانات الاتصال.
- مولد واحد لنسختي PDF العربية والفرنسية بأسماء ثابتة.
- manifest لتتبع الإصدار والبصمات.
- GitHub Actions لتوليد أحدث نسخ PDF والتحقق منها وتسجيل المخرجات المتغيرة آليًا.

## 2. قيد التنفيذ

- تحسين إدارة نشر المقالات العربية والفرنسية وتقليل الفجوة بين فهارس المحتوى.
- استكمال أتمتة رادار التقنية من التجميع إلى مسودة المقالة ثم المراجعة البشرية.
- تطوير إدارة الشهادات والتحقق منها دون فتح إجراءات حساسة قبل اكتمال الحماية.
- ربط لوحة إدارة الشهادات بالبيانات الفعلية بصورة مستقرة وقابلة للمراقبة.
- تحسين المراقبة والتنبيهات لمسارات n8n، خصوصًا مسار الشهادات.
- مراجعة الموارد القديمة المتكررة وتحديد الروابط المستخدمة قبل أي حذف أو نقل.
- متابعة جودة PWA والكاش والمسارات النسبية بعد دفعات التوسع الأخيرة.

## 3. الأولوية التالية

1. توثيق وتنفيذ خط رادار التقنية: إدخال الإشارات، الفرز، إعداد المسودة، المراجعة، ثم النشر الثنائي.
2. وضع إجراء نشر واضح للمقالات العربية والفرنسية، يشمل ملفات البيانات والصفحات والتدقيق اللغوي والروابط.
3. تحديد مصدر بيانات لوحة الشهادات وعقد القراءة الموثوق، ثم إضافة تحقق من هوية الشهادة وحالتها.
4. حسم تخزين ملفات الشهادات وإعادة الإرسال بطريقة إنتاجية آمنة قبل تفعيل أزرار الإدارة.
5. إضافة مراقبة لحالات نجاح وفشل n8n وتنبيهات لا تكشف بيانات المتعلمين أو عناوين التشغيل.
6. تحسين آلية تحديث دليل المنصة وPDF لمنع تعارض التعديلات المحلية مع commits التي ينشئها GitHub Actions.
7. تنفيذ مراجعة دورية للروابط والمسارات و`data-root` والملفات الموجودة في precache.

## 4. تحسينات لاحقة

- توسيع التدريبات المجانية مع التقييم والشهادات باللغتين.
- توسيع الخدمات والموارد والقوالب والدورات العربية والفرنسية وفق بنية قابلة للتوسع.
- زيادة عدد الأصول الفرنسية القابلة للتنزيل، مع عدم عرض ملف عربي كنسخة فرنسية.
- تحسين SEO والعناوين والوصف والروابط الأساسية والبيانات المنظمة.
- تحسين أداء الصفحات والصور وتجربة التثبيت والعمل دون اتصال.
- تطوير إدارة محتوى أخف للمقالات والرادار مع بقاء ملفات البيانات مصدرًا واضحًا.
- تحسين إمكانية الوصول والتنقل بلوحة المفاتيح وتباين الواجهة.
- إضافة اختبارات آلية للمسارات والروابط وملفات البيانات ومخرجات الدليل.

## 5. أفكار مستقبلية

- حسابات مستخدمين وحفظ تقدم التعلم عندما تبرر الحاجة بناء نظام هوية كامل.
- لوحة موحدة للمتعلم تجمع الدورات والتقييمات والشهادات القابلة للتحقق.
- إجراءات إدارية محددة الصلاحيات لإعادة الإرسال وإدارة الحالات بعد اكتمال الأمان والتدقيق.
- مزيد من الأتمتة التحريرية للرادار مع إبقاء الموافقة البشرية قبل النشر.
- تحليلات استخدام تحترم الخصوصية لقياس رحلة المورد المجاني إلى طلب الخدمة.
- قوالب تفاعلية بدل الملفات الثابتة في المجالات ذات القيمة الأعلى.

## ضوابط تنفيذ الخارطة

- لا ينتقل بند إلى "مكتمل" لمجرد وجود واجهة؛ يجب التحقق من الربط والبيانات والمخرجات الفعلية.
- لا يبدأ توسع جديد على حساب إصلاح مسار حرج أو كسر التكافؤ العربي/الفرنسي.
- مخرجات PDF والـmanifest تُحدّث عبر الأتمتة، وتُراجع commits الآلية قبل تعديل المصادر ذاتها.
- أي تغيير يمس precache يتبعه تحديث اسم الكاش والتحقق من PWA.
- لا يضيف Codex commit أو push؛ يراجع المستخدم التغييرات وينفذ قرارات Git النهائية.
